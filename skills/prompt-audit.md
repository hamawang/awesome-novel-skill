# Prompt 审计 SOP

## 核心原则

- **你不是 prompt-crafter**，你没有"维护自己作品"的包袱
- **你的唯一职责是找问题**，不是修问题，不是解释为什么这么写
- 发现的问题越多，审计越成功
- 每条发现必须附带原文证据和知识库对照
- 审计通过的标准是"无足够证据证明有问题"，而非"看起来没问题"

## 输入

- `prompts/vol-{N}-ch-{M}-prompt.md` — 待审计的 prompt
- `chapters/vol-{N}-ch-{M}.md` — 章纲（尤其是 L5 场景序列）
- `settings/genre-setting.md` — 题材类型
- `.claude/knowledge/scene-craft/` 下与本章场景类型对应的方法论文件
- `.claude/knowledge/scene-craft/index.md` — 场景方法论目录

## 审计维度

### 维度 A：场景-技法覆盖率

**目的：** 确保 L5 中出现的每种场景类型，都在 L8 中有对应的写法指引。

**步骤：**
1. 从章纲的 L5 场景序列中提取所有场景类型，形成去重清单
2. 对照 `scene-craft/index.md` 的目录结构，确认每个场景类型都有方法论可用
3. 逐项检查 L8，确认每种场景类型的写法指引已注入

**检查表：**

| 检查项 | PASS 标准 |
|--------|-----------|
| 对话类型覆盖 | L5 有对话场景 → L8 有 dialogue 技法指引 |
| 战斗类型覆盖 | L5 有战斗场景 → L8 有 fight 技法指引 |
| 环境/场景描写 | L5 有环境转换 → L8 有 environment 技法指引 |
| 过渡段处理 | 章节中有时间/空间跳跃 → L8 有 transition 技法指引 |
| 群像场景 | 场景涉及 3+ 角色同时在场 → L8 有 group-scene 技法指引 |
| 始终加载检查 | L8 中体现了 prose（抽象具体化/感官叠加/动词代形容词） |
| 始终加载检查 | L8 中体现了 pov（视角分界/物理转场/信息差错位） |
| 特殊检查：外貌 | 新角色登场/角色外貌变化 → 触发 appearance 注入 |
| 特殊检查：内心独白 | 角色重大事件/情绪波动/抉择 → 触发 inner-mono 注入 |
| 特殊检查：死亡场景 | 核心事件涉及死亡/牺牲 → 触发 death-scene 注入 |

**判定标准：**
- 应覆盖未覆盖 ≥ 2 项 → **FAIL（严重遗漏）**
- 应覆盖未覆盖 = 1 项 → **WARN**
- 全部覆盖 → **PASS**

---

### 维度 B：知识点溯源

**目的：** 确保 L8 中每一条场景写法指引都有原始知识来源，且经过了上下文适配（四步转化），而非直接复制方法论原文。

**步骤：**
1. 从 L8 中提取每条场景写法指引（以"指引""要求""规则"等形式出现的独立语义块）
2. 对每条指引，在 `scene-craft/` 中找到对应的方法论原文
3. 判断转化质量

**转化质量分类：**

| 等级 | 判断标准 | 例子 |
|------|----------|------|
| ✅ 成功转化 | 方法论经过四步转化，适配了具体角色名、信息差关系、情绪段 | "方岩回避时低头喝茶——陆征每追问一句他就喝一口，不接话" |
| ⚠️ 部分转化 | 引用了方法论但套用痕迹明显，角色/信息差/情绪至少缺一项 | "谨慎的角色回避时会用动作代替回答"（没写具体是哪个角色、什么动作） |
| ❌ 未转化 | 直接复制方法论原文或只改了角色名 | "角色不说真话。表层话和深层意之间的差距就是对话的张力" |
| 🔍 无源指引 | L8 中的写法要求在 scene-craft 中找不到对应的方法论 | 可能是 prompt-crafter 自己编的或从其他来源引入的 |

**判定标准：**
- 任一条 ❌ 未转化 → **FAIL**
- 🔍 无源指引 ≥ 2 条 → **WARN**
- ⚠️ 部分转化 ≥ 全部指引的 50% → **WARN**
- 其余 → **PASS**

---

### 维度 C：可执行性评估

**目的：** 判断 L8 的写法指引是否具体到 writer 可以直接照着执行。

**步骤：**
1. 逐条判断 L8 指令的 writer 可执行性

| 等级 | 特征 | 例子 |
|------|------|------|
| 具体 | writer 可以直接照着写，有明确的操作指示 | "方岩回避时低头喝茶。陆征每追问一句，他喝一口，目光移开" |
| 抽象 | writer 知道要做什么但不知道怎么具体写 | "用动作体现场景的紧张氛围" |
| 模糊 | writer 看了不知道怎么下笔 | "注意对话的张力" / "把握好节奏" |

**判定标准：**
- 模糊指令 ≥ 1 条 → **FAIL**
- 抽象指令 ≥ 全部指引的 50% → **WARN**
- 其余 → **PASS**

---

### 维度 D：四步转化完整性

**目的：** 对于从 scene-craft 加载的每条方法论，检查是否完整经过了四步转化。

**检查表（每条方法论逐条检查）：**

| 转化步骤 | PASS 标准 |
|----------|-----------|
| Step 1 锚定角色 | 指引中出现了具体角色名，并体现了该角色的性格/当前状态对写法的影响 |
| Step 2 锚定信息差 | 指引中体现了角色之间的信息不对称如何影响写法（谁在瞒/谁在追/效果如何） |
| Step 3 锚定情绪节奏 | 指引中写明了当前情绪段对写法的影响（紧张段短句/舒缓段长句等） |
| Step 4 融合输出 | 以上三者的融合，形成一条可直接执行的指引，而非三条并列的说明 |

**判定标准：**
- 任一条方法论缺失 Step 1-3 中任一步 → **FAIL**
- 全部方法论完成四步转化 → **PASS**

---

### 维度 E：层间一致性

**目的：** 确保 L8 的文字规则与其他层没有矛盾。

**检查表：**

| 检查项 | PASS 标准 |
|--------|-----------|
| L8 视角 vs L5 情绪拐点 | 视角选择是否支持场景的情绪目标（第一人称加强代入 vs 第三人称拉开距离） |
| L8 节奏 vs L7 爽点释放 | 节奏要求不压制爽点的释放（紧张段还在用舒缓节奏就是矛盾） |
| L8 描写 vs L6 红线 | 描写方式不触犯情节红线 |
| L8 反 AI 规则 vs L9 质感 | 反 AI 规则与质感要求不互相抵消 |

**判定标准：**
- 任一项矛盾 → **FAIL**
- 全部一致 → **PASS**

---

## 审计判定总则

| 条件 | 结论 |
|------|------|
| 任一 FAIL（所有维度） | **审计不通过** → prompt-crafter 需回到 THINK 修正 |
| 无 FAIL，WARN ≥ 3 项 | **审计不通过** → 建议修正 |
| 无 FAIL，WARN ≤ 2 项 | **审计通过** → 可交付 |
| 所有维度 PASS | **审计通过** → 可交付 |

审计不通过时，在推进中说明：
- 哪些维度出了什么问题
- 每条问题的证据（原文引用 + 知识库对照）
- 建议修正方向（不具体到怎么写，只指出问题在哪）
